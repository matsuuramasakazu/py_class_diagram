# Implementation Plan - Issue #5: 関係線の高度化

## 1. 要件分析と戦略

### 1.1 現状の課題と目標
現在の関係線は単純な直線であり、複雑な図において線が重なったり、見栄えが悪かったりする。
Issue #5 では、ベジェ曲線による滑らかな描画、制御点による手動調整、および自己参照リレーションのサポート等、高度な描画機能の実装が求められている。また、ドメインモデル層での計算ロジック実装と標準ライブラリのみの使用という制約がある。

### 1.2 技術的アプローチ
-   **データモデル拡張 (`model.py`)**:
    -   `UMLRelationship` クラスを拡張し、制御点（Control Points）の情報を保持できるようにする。
    -   ただし、永続化（`persistence.py`）との兼ね合いを考慮し、後方互換性を維持する。
-   **ドメインロジック (`model.py` / `geometry.py`)**:
    -   座標計算ロジック（ベジェ曲線、交点計算、制御点位置の自動計算）を実装する。要件に従い `model.py` または新規モジュールに配置する。
    -   3次ベジェ曲線 $B(t) = (1-t)^3 P_0 + 3(1-t)^2 t P_1 + 3(1-t) t^2 P_2 + t^3 P_3$ の計算式を実装。
-   **描画ロジック (`rendering.py`)**:
    -   `tkinter` の `create_line` (smoothオプションなし、またはポリライン近似) を使用してベジェ曲線を描画する。
    -   矢印の向き計算を曲線の接線ベクトルに基づいて行うように修正する。
    -   自己参照リレーション（ループ）の特異的な描画ロジックを追加する。
-   **インタラクション (`editor_canvas.py`)**:
    -   関係線の選択機能。
    -   制御点の表示とドラッグ操作の実装。
    -   端点のドラッグ＆ドロップによる接続先変更（リワイヤリング）。
    -   クラス間のドラッグ＆ドロップによる新規リレーション作成UXの改善（最短辺の中点自動選択）。

## 2. 実装ステップ

### Step 1: 幾何計算モジュールの作成
-   **Target**: `geometry.py` (新規作成)
-   **Action**:
    -   ベジェ曲線計算関数（tにおける座標取得）。
    -   ベジェ曲線の接線ベクトル計算（矢印の向き用）。
    -   矩形と曲線の交点計算（厳密解は難しいので、分割線分との交差判定で近似するか、二分探索等の数値解法を使用）。
    -   最短辺の中点計算ロジック。

### Step 2: データモデルの拡張
-   **Target**: `model.py`, `persistence.py`
-   **Action**:
    -   `UMLRelationship` に `source_handle` (P1), `target_handle` (P2) などの制御点オフセット情報を追加。または `control_points` リスト。
    -   `persistence.py` でこれらの追加情報をJSONの `layout` セクション等に保存・復元できるようにする。

### Step 3: 関係線生成・更新ロジックの実装
-   **Target**: `model.py`
-   **Action**:
    -   関係線作成時にデフォルトの制御点（直線上の10%位置など）を初期化するロジック。
    -   複数本ある場合のオフセット自動計算ロジック。

### Step 4: 描画ロジックの刷新
-   **Target**: `rendering.py`
-   **Action**:
    -   `draw_relationship_line` をベジェ対応版に書き換え。
    -   自己参照リレーション用の描画パス生成ロジック追加。
    -   矢印描画位置と回転角の計算をベジェ曲線の終端接線に合わせる。

### Step 5: インタラクションの実装（Canvas）
-   **Target**: `editor_canvas.py`
-   **Action**:
    -   **作成**: クラスD&Dでの作成と、最短辺自動判定。
    -   **編集**: 関係線選択時に制御点ハンドル（小さな矩形や円）を表示。
    -   **操作**: ハンドルをドラッグして `UMLRelationship` の制御点データを更新し、再描画。
    -   **リワイヤリング**: 端点ハンドルをドラッグして別のクラス上でドロップした際、`source` / `target` を更新する。

## 3. 検証計画 (Test Plan)

### 手動検証手順
1.  **通常リレーション**:
    -   クラスAからクラスBへドラッグ＆ドロップで線を作成。最短辺同士が結ばれること。
    -   線を選択し、制御点をドラッグして曲線に変形できること。
    -   曲げた状態で矢印の向きが線の接線方向と合っていること。
2.  **自己参照**:
    -   クラスAをクリック＆ドロップ（または自分自身にドロップ）で自己参照線が作成されること。
    -   右下を回るような形状で描画されること。
3.  **複数線**:
    -   同じクラス間に2本目の線を引いた際、自動的にオフセットされて重ならないこと。
4.  **リワイヤリング**:
    -   A->Bの線のB側端点をドラッグし、クラスCにドロップするとA->Cに変更されること。
